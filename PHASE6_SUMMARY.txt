USAGE WORKFLOWS
================================================================================

WORKFLOW 1: Fresh Installation (Most Common)

  Host $ ./tools/run.sh --disk-only
  Host $ ./tools/run.sh --build --disk
  VM   $ sudo ./tools/install_sunlightos.sh /dev/sda
  [Interactive prompts for hostname, timezone, users]
  VM   $ sudo poweroff
  Host $ [System now installed on disk]

WORKFLOW 2: Rebuild and Test

  Host $ ./tools/run.sh --build --disk
  VM   $ sudo ./tools/install_sunlightos.sh /dev/sda
  VM   $ sudo poweroff
  Host $ qemu-system-x86_64 -drive file=target/sunlightos-disk.img,format=raw

WORKFLOW 3: Disk-Only Preparation

  Host $ ./tools/run.sh --disk-only
  [Disk ready for later use]
  Host $ ./tools/run.sh --build --disk --vnc
  [VNC headless boot with disk attached]

CONFIGURATION FILES GENERATED
================================================================================

/etc/hostname
  Simple text file containing system hostname
  Example: "myhost"

/etc/localtime
  Symlink or copy of zoneinfo file
  Example: → /usr/share/zoneinfo/Asia/Tehran

/etc/auth/users.json
  JSON structure with user accounts and hashed passwords
  Format:
    {
      "version": 1,
      "users": [
        { "uid": 0, "username": "root", "password_hash": "...", ... },
        { "uid": 1000, "username": "alice", "password_hash": "...", ... }
      ],
      "groups": [...]
    }

/etc/fstab
  Mount configuration with partition UUIDs
  Example:
    UUID=abc-123  /     ext4  defaults  0  1
    UUID=def-456  /boot vfat  defaults  0  2

TECHNICAL SPECIFICATIONS
================================================================================

DISK IMAGE:
  Format:         Raw (5GB)
  Type:           Virtual hard disk
  Location:       target/sunlightos-disk.img
  Partition 1:    512 MB FAT32 (bootable)
  Partition 2:    ~4.5 GB ext4 (root filesystem)
  MBR:            Limine bootloader installed

INSTALLER SCRIPT:
  Language:       Bash (POSIX sh compatible)
  Lines:          420
  Dependencies:   fdisk, mkfs.vfat, mkfs.ext4, blkid, mount, rsync
  Privileges:     Requires root (sudo)
  Runtime:        ~40-70 seconds (including filesystem clone)

INPUT VALIDATION:
  Hostname:       RFC 952 compliant (a-z0-9-, max 63 chars)
  Timezone:       Checked against /usr/share/zoneinfo/
  Username:       POSIX rules (a-z_, alphanumeric, max 32 chars)
  Password:       Non-empty, SHA256 hashed (no salt)

FILESYSTEM CLONE:
  Tool:           rsync
  Source:         Live RAMFS (/)
  Destination:    Mounted ext4 partition
  Excludes:       proc, sys, dev, run, tmp, var/tmp, var/lock, boot, config
  Typical size:   ~100-200 MB cloned data
  Time:           ~30-60 seconds on QEMU

COMMAND REFERENCE
================================================================================

1. CREATE DISK IMAGE (HOST)
   $ ./tools/run.sh --disk-only
   Creates: target/sunlightos-disk.img (5GB raw)

2. BOOT WITH DISK (HOST)
   $ ./tools/run.sh --build --disk
   Rebuilds kernel and launches QEMU with disk attached

3. INSTALL SYSTEM (INSIDE VM)
   $ sudo ./tools/install_sunlightos.sh /dev/sda
   Interactive wizard prompts for all configuration

4. BOOT INSTALLED DISK (HOST)
   $ qemu-system-x86_64 -drive file=target/sunlightos-disk.img,format=raw

5. SHOW HELP (HOST)
   $ ./tools/run.sh --help
   Displays updated help with --disk documentation

SUPPORT & TROUBLESHOOTING
================================================================================

For detailed troubleshooting, see PHASE6_QUICKSTART.md

Common Issues:
  Q: Disk not visible in VM
  A: Restart with ./tools/run.sh --disk flag

  Q: Partitions not detected
  A: Script retries automatically; if fails, run partprobe manually

  Q: Installer requires sudo but I'm root
  A: Still run with sudo ./tools/install_sunlightos.sh

  Q: Invalid hostname/timezone rejected
  A: Check format against RFC 952 / /usr/share/zoneinfo/

  Q: Installation stops during clone
  A: Ensure /mnt has sufficient free space
