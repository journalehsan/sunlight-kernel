┌─ KEY ENHANCEMENTS ────────────────────────────────────────────────────────────┐
│                                                                               │
│  1. Intelligent Priority Calculation                                         │
│     • Combines tier (0/10/20) + nice (-20..19) + interactive_bonus          │
│     • effective_priority() method: lower value = higher priority             │
│     • Example: High tier (0) + nice 0 + bonus 20 → eff_prio = -2            │
│                                                                               │
│  2. Adaptive Time Slicing                                                    │
│     • dynamic_quantum() considers nice and burst_score                       │
│     • Interactive (low burst): 2-40 ticks (longer slices)                   │
│     • CPU-bound (high burst): shorter slices for responsiveness             │
│                                                                               │
│  3. Priority-Ordered Queues                                                  │
│     • Processes sorted by effective_priority within each tier               │
│     • insert_by_priority() maintains sorted order                            │
│     • Prevents priority inversion                                            │
│                                                                               │
│  4. Enhanced Logging                                                         │
│     • All process creation logs show: burst_score, tier, nice, eff_prio     │
│     • Example: [SCHED] CREATED process #1 'init' idx=0 burst_score=256      │
│                tier=High nice=0 eff_prio=-2                                  │
│                                                                               │
